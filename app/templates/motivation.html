Introduction

Facial recognition systems match a digital image or video frame against a database of faces. By pinpointing and measuring facial features from an image, a system like this is used to authenticate users. There are a wide variety of attributes that can be classified based on the appearance of an individual's face, such as a beard, a hat, etc. Facial attribute classification involves classifying various aspects of a facial image.

You are about to interact with a demo that is designed to demonstrate the logic and potential dangers of facial recognition technologies, following a flow with determined face recognition algorithms and a progressive survey. The demo works through deep learning models that have been trained using large data sets.

Motivation

This demo aims to inform about what Facial Recognition Technologies (FRT) are, how they work, where they are currently used and where potential dangers lie. It conveys these dangers and discussions accepted by various non-governmental organizations to the user with a didactic approach.

It does so by demonstrating FRT in action, explaining underlying science and asking questions. After every layer of experience and knowledge the questions repeat.

Architecture

Underlying tech:

The demo is based on the python programming language and two significant frameworks. One for detecting faces on a dynamically changing image and one for analyzing facial attributes

OpenCV

Faces in an image were detected using the OpenCV Framework. This framework was used to detect distinct faces utilizing the "haar cascade-frontal face"-Classification.

DeepFace

Deepface is a lightweight framework for Python that analyses age, gender, emotions, and other facial attributes. Many models, including VGG-Face, Google FaceNet, OpenFace, Facebook DeepFace, DeepID Dlib, and SFace, are warped by this framework. The "Facebook DeepFace" model, however, was the one we employed to analyse the facial attributes in the upcoming demo.
